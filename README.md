# Anti-AI Design System

> A design system whose product **is** removing the cara-de-IA (AI face) from web apps generated by Lovable, v0, Cursor, and Claude. One opinionated preset, ten anti-pattern rules, seven identity gestures, and a coherence checklist that codifies what trident catches and what your eye still has to.

This isn't a brand kit ‚Äî it's **structural infrastructure**. It doesn't tell you what your brand looks like. It tells you which of the 10 default decisions the AI made for you that you need to overwrite so your brand can show up.

## CONTENT FUNDAMENTALS

The voice of this system is **direct, confident, and slightly brusque** ‚Äî it's the voice of someone who has seen 200 generated dashboards and is tired of explaining why they all feel the same.

### Tone & casing

- **Lowercase headings** in code/UI ("anti-checklist", "page title", "section headings"). Title Case only for proper nouns and document chapter titles.
- **No exclamation points.** Confidence comes from precision, not enthusiasm.
- **No marketing softeners.** "Should consider" ‚Üí "do not". "Might want to" ‚Üí "use". "Recommended" ‚Üí "regra".
- **PT-BR + English mix in source.** The upstream docs are PT-BR; English versions exist for distribution. Never apologize for either.

### Sentence shape

- **Short declarative.** "Card soup is the tell." not "One thing we've noticed is that the use of Card patterns can sometimes signal..."
- **Verbs over nouns.** "Decide your radius" not "the radius decision".
- **Numbers > adjectives.** Not "small delta" ‚Äî "delta < 5%". Not "many violations" ‚Äî "5+ of these = cara de IA".

### Examples directly from the system

- > "Card soup. Each block uses the same Card pattern, creating visual monotony."
- > "Container fluid with generic padding. `container mx-auto` is the lorem ipsum of layout."
- > "If 5+ of these apply, it has cara de IA."
- > "**NUNCA** `<Card>/<CardHeader>/<CardContent>` do shadcn."
- > "Brackets were meant to REPLACE icons; having both is mechanical rule-following without understanding."

### I vs. you

- Docs use **imperative ("Fa√ßa‚Ä¶ N√£o fa√ßa‚Ä¶")** not "I recommend" or "you should". The system tells; it does not persuade.
- When second person is used, it's `voc√™` (PT) or `you` (EN), never `we`.

### Emoji policy

**No emoji in UI.** No emoji in docs body either. Allowed: ‚úÖ/‚ùå in checklists when scanability matters, and never decorative.

### Vibe

Workshop notebook of an opinionated technical lead, not a polished marketing site. Code blocks are the primary unit of explanation. If a paragraph is doing the work that a 6-line code snippet would do clearer, replace it.

## VISUAL FOUNDATIONS

### Color

- **Foreground always carries hue.** `S > 0`. Pure `0 0% L%` is forbidden ‚Äî it's the #1 tell of a generator. The `default` preset uses `16 38% 12%` (dark cocoa).
- **Background carries the same hue family** as foreground (within ¬±30¬∞ of H). Cool foreground + warm background = dissonance.
- **Status colors** are desaturated and temperature-aligned. Never `120 100% 50%` (neon green) or `0 100% 50%` (pure red) for `success`/`destructive` ‚Äî the `default` preset brings `success` to `152 85% 30%`.
- **Sidebar uses a different surface**, not a tint. The `default` preset makes it teal `184 100% 18%`. Delta ‚â• 10% L from main, OR opposing hue family.

### Type

- **Two families.** Body + display. Never three.
- Page title is `<h2>` at `text-xl` `font-semibold` (`20px / 600`). **`<h1>` is reserved for landing/marketing only.**
- Section title is `<h3>` at `text-sm` `font-medium`.
- Body is `text-sm` (`14px`). Default is dense by AI standards ‚Äî that's the point.
- **Never `font-bold` (700).** `font-semibold` (600) is the ceiling.
- **No `tracking-wide`.** It reads as "fancy" and is a tell. `tracking-tight` is allowed on display headers.
- Mono (`Geist Mono`) is reserved for tabular numbers and code only.

### Spacing

- Religious short scale: **`1 / 1.5 / 2 / 4 / 5 / 6 / 8`** (in 4px increments). **Never** 3, 7, 9.
- Container vertical = `space-y-5` (20px), nothing else.
- Shell padding = `p-6` (24px) ‚Äî applied at the layout, **never** nested in children.
- Form: `space-y-1.5` within field, `space-y-4` between fields.

### Backgrounds

- **No gradients in app UI.** Hero sections on landing pages may use them; the rest may not. Specifically banned: bluish-purple AI gradients.
- **No full-bleed photography in app shells.** Photos go inside content cards, with explicit aspect ratio.
- **No repeating patterns or textures.** A flat, hue-washed surface is the canvas. Decoration comes from typography and from **one** accent color.
- The sidebar is the most decorated surface in the app. Treat it as the brand's contact patch.

### Animation

- **150‚Äì300ms only.** `duration-150` for hover (color, opacity). `duration-200` for state. `duration-300` for layout shift.
- Easing: Tailwind default `cubic-bezier(0.4, 0, 0.2, 1)`. **No `ease-bounce`, no springs.** No `duration-500+`.
- `transition-colors` for hover; `transition-all` is a code smell.

### Hover states

- Buttons: `hover:bg-accent` with delta L ‚â• 10% from base, OR border swap. **If hover is invisible at squint distance, it failed.**
- Rows in lists: `hover:bg-muted/50` (alpha) ‚Äî softer than a button hover.
- Links inline: underline appears on hover (not always-on).

### Press / active states

- Buttons: no shrink, no scale. **Color shift only.** `active:bg-accent/80`.
- Active nav item: a **gesture**, not a bg change. Vertical bar, dot, border-l accent. See `06-identity-gestures.md` rule #2.

### Borders

- One `--border` token, with hue. The `default` preset uses `30 20% 87%`.
- **Sidebar borders use `--sidebar-border`**, not `--border`. They're a different surface.
- Adjacent elements (sidebar header touching main header) **must share** the border token, or the seam is visible at 200% zoom.

### Shadows

- `default`: `shadow-sm` for static cards, `shadow-md` for hover. `shadow-lg` only for popovers/modals. **No `shadow-xl` or `2xl`.** Over-elevation is cara de IA.

### Capsules vs. protection gradients

- **Capsules.** Status badges and pill counts. Always `rounded-full`, padded `px-2 py-0.5` or `px-3 py-1`. The `default` preset uses semi-transparent bg (`hsl(var(--success) / 0.1)`).
- **Protection gradients.** Banned. If you need a label to read over imagery, put it on a solid bg, not a gradient overlay.

### Layout rules (fixed elements)

- **Sidebar width is intentional.** The `default` preset ships `272px` expanded / `72px` collapsed. Never `w-60` / `w-64` (Tailwind defaults).
- **Header height matches sidebar header height** to the pixel ‚Äî that border line is one continuous line crossing the seam. (Coherence rule #1.)
- **Page is `max-w-7xl` (80rem)** for app, `max-w-5xl` for editorial, `max-w-xl` for forms. Never `container mx-auto`.

### Transparency & blur

- Use sparingly. `bg-foreground/5` for subtle dividers and `bg-muted/50` for row hover are fine.
- **No `backdrop-blur` for decoration.** Reserved for sticky headers over scrolling content where blur is functional.

### Imagery

- When this system needs imagery, it's **warm cocoa-toned**. Reject saturation cranked to 100%. Source photos should sit at `saturate(0.85)` or lower.
- Illustrations: hand-drawn line art only, never AI-generated stock. If you don't have illustrations, use **typography as the visual instead**.

### Corner radii

- `default`: 6 ‚Üí 8 ‚Üí 12 ‚Üí 16 ‚Üí 20 (sm/md/lg/xl/2xl). Buttons `lg`, rows `xl`, cards `2xl`. Variable scale. The exception is `rounded-full` for avatars and capsules.

### Cards

- `default` card: `bg-card rounded-2xl border border-border shadow-sm`, `px-5 py-3` header divided by `border-b`, `px-5 py-4` content.
- **Never** the shadcn `<Card>/<CardHeader>/<CardContent>` triplet. Always raw divs with the classes above. (Anti-pattern #1.)

## ICONOGRAPHY

The system's stance: **icons earn their place through information, not decoration.** The default count of icons in any view is zero. Each icon must answer "what would be lost if this were just text?"

### Approach

The `default` preset uses **Lucide React** (stroke-based, 1.5px weight, rounded joins). It matches the soft radii. Lucide is also what Lovable defaults to, so the substitution cost is zero.

### When to icon

- ‚úÖ Status badges (success / warning / error indicator dot).
- ‚úÖ Primary action button when the verb is unambiguous (`<Save />` Save).
- ‚úÖ File type marker (PDF, image, video).
- ‚úÖ Sidebar nav items (kept short and consistent).

### When NOT to icon

- ‚ùå Form labels.
- ‚ùå Secondary buttons.
- ‚ùå Section titles inside cards.
- ‚ùå Every nav item just because the others have one.
- ‚ùå As pure decoration in hero sections.

### Source

This project ships:
- `assets/icons/` ‚Äî a small set of curated Lucide SVGs (search, settings, plus, check, x, chevron-right, more-horizontal, file, image, alert-triangle, info, copy, external-link, user). Use these directly or load from CDN.
- For broader needs, link Lucide from CDN: `https://cdn.jsdelivr.net/npm/lucide-static@latest/icons/<name>.svg`.

**Substitution flag:** the upstream repo doesn't ship icon assets ‚Äî it relies on `lucide-react` as a peer dependency. We've copied a representative subset of Lucide SVGs as static assets so HTML mocks can use them without a build step. **Confirm with us if you'd prefer a different icon family** (Phosphor, Heroicons, Tabler).

### Emoji & unicode

- **No emoji** in UI.
- **Unicode is allowed** when it carries meaning: `‚ñ≤ ‚ñº ‚úì ‚úï ‚Üí ‚Üê` for sort, status, navigation. Use sparingly ‚Äî the `default` preset prefers Lucide for icon-shaped affordances.

## How to use

1. Copy `colors_and_type.css` into your project (or copy `presets/default/tokens.css` from the upstream repo).
2. Tokens are scoped to `:root` ‚Äî they apply automatically; no `data-preset` attribute needed.
3. Use the structural rules in `docs/02-structural-rules.md` and the anti-checklist in `docs/01-anti-patterns.md` to audit every screen before you ship it.
4. Run the coherence checklist (`docs/05-coherence-checklist.md`) and identity gestures pass (`docs/06-identity-gestures.md`) before "looks done".

## Caveats & known substitutions

- **Fonts:** all four font families (Poppins, Lora, Inter, Geist Mono) load from Google Fonts CDN. Local `.ttf` fallbacks under `fonts/` are placeholders ‚Äî bring real font files if you need offline rendering.
- **Icons:** Lucide is treated as the canonical set. If your project standardizes on another family, swap the SVGs in `assets/icons/` and update this README's ICONOGRAPHY section.
- **Dark mode:** light-only by default in both presets. Dark mode is per-project work, not standardized here.
- **Translations:** the upstream docs in `docs/` are PT-BR. Re-reading a doc's *intent* in English from this README's summaries is supported; line-by-line EN translations are not in scope.
