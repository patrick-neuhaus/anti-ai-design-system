Filosofia

Por que a maioria dos apps gerados parece o mesmo template SaaS branco com gradiente roxo?

Porque agentes recombinam padrões mais comuns. Sem regras explícitas anti-padrão, todo app vira variação do mesmo arquétipo.

— Manifesto, README.md

Anti-padrão #1

Cores hardcoded

Componentes que bakeiam #0066FF direto no JSX viram dívida no primeiro rebrand. Sempre via hsl(var(--*)).

Anti-padrão #2

Mega-prop

Componente com 30 props vira API que ninguém lembra. Composição via slots (children, leading, trailing) ao invés de configuração total.

Anti-padrão #3

Brand bakeado

Sidebar, Login, StatCard que importam logo da marca direto. Brand chega via prop, não import. Senão o sistema não migra.

Anti-padrão #4

Neutros sem hue

Toda escala de cinza com 0% saturação é a marca registrada de UI gerada. Neutros carregam temperatura — warm, cool, deep.

Sistema de tokens

Variáveis semânticas, não literais.

Cada cor, espaço e radius é declarado como papel (primary, accent, muted) — não como valor (#hex, px). Trocar tema = trocar variáveis, não código.

Como usar

Três caminhos. Mesmos tokens.

Use no Lovable como Memory + tokens, em qualquer projeto React copiando o kit, ou só HTML/CSS plano via foundations.

// 1. Cole tokens.css em src/styles/design-tokens.css // 2. Importe em src/index.css: @import "./styles/design-tokens.css"; // 3. Use via classes ou tokens diretos: <div className="bg-primary text-primary-foreground"> Conteúdo </div> // 4. Cole memory files em .lovable/memory/design/ // 5. Lovable Agent passa a respeitar regras automaticamente
